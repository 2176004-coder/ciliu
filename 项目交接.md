# 词流（Lexis）项目交接说明

把这份发给新对话当背景。这是一个**已经做到 v3.0 的、可用的本地英语阅读+背单词网页应用**，后续多是迭代改进。

## 一句话
给中文母语者用的英语学习工具：读英文文章 → 点词查离线词典 → 攒生词 → 多种模式复习。**纯前端、离线、双击即用、无需构建/服务器。**

## 文件位置
都在 `/Users/jiangyanming/Desktop/lexis/lexis-app/`：
- **`index.html`** — UI 外壳 + 全部 CSS。双击它即可使用。
- **`app.js`** — 全部逻辑，约 1300+ 行，原生 JS（ES5 风格，`var` + 函数声明，无框架）。
- **`ecdict.js`** — 词典数据，**几十 MB 的巨型文件，千万不要整篇 Read**（约 700 万 token）。要查内容用 `Grep` 配 `-o` 抽样。
- **`制作完整词典.html`** — 词典生成器（把 ECDICT 的 ecdict.csv 转成 ecdict.js）。
- `lexis.jsx`（原始 React 原型，仅作参考）、`lexis-cowork-brief.md`（最初需求）。

## 架构（重要）
- 单一全局 `state` 对象；`render()` 整体重建 `#app` 的 innerHTML；每次 render 后 `bind()` 重新挂事件监听。改任何交互都遵循这个 render/bind 模式。
- 顶部三个标签：**阅读 / 词汇 / 复习**。
- 数据存 localStorage：`lexis_articles`、`lexis_vocabulary`、`lexis_known`、`lexis_autospeak`。
- 词典：`window.ECDICT_RAW`（制表符分隔，字段 `word\tphon\tpos\tfreq\ttag\ttrans`，兼容旧的 4/5 段）解析成 `DICT` Map；`window.LEMMA_RAW`（变形→原形）+ 内置 `IRREGULARS` 不规则表 + 后缀规则做原形还原（`lookupWord` / `resolveLemma`）。
- 设计风格：暖色纸张感，衬线字体（Fraunces / Crimson Pro / Noto Serif SC），颜色都在 `:root` 的 CSS 变量里。版本号字符串在 app.js 顶部（当前 **v3.0**），每次改动顺手 +0.1。

## 功能现状
- **阅读**：粘贴或导入文章；正文逐词可点；点词弹**贴词浮窗**。词三态：蓝=新词、绿=生词、无色=已掌握。浮窗按钮：绿色【加入生词】主 + 灰色【我已掌握】次；关闭浮窗默认存为生词。
- **发音**：Web Speech API（`speak`），顶部有自动朗读开关；选音逻辑会避开 macOS 的搞怪语音。
- **词汇**：子标签 生词/已掌握/**词书**；搜索/添加用分段开关切换；排序（最近/常见度/字母）；批量勾选操作；行内有考试等级小徽章。**词书**来自 ECDICT 的考试标签（cet4/cet6/ky/ielts/toefl/gre/gk/zk）。
- **复习**：五种模式——单词卡(flash，认识→移入已掌握/不认识→留着)、选择题(mc)、例句挖空(cloze)、配对游戏(match)、综合测验(test，10 题给分)。复习用的是**用户自己读到的原文例句**。
- **导入文章**：txt（离线）、PDF（pdf.js）、EPUB（JSZip），后两者首次导入时联网从 CDN 懒加载解析库；已有内容时**追加**而非覆盖。
- **编辑/续写**：阅读某文章时右上角「编辑/续写」可改标题正文、继续导入追加。
- **备份**：顶部导出/导入 JSON（含文章、生词、已掌握）。

## 词典再生成
若缺词频或标签，需用 `制作完整词典.html` 重跑：到 ECDICT 的 GitHub Releases 下载 `ecdict-csv-28.zip` → 解压得 `ecdict.csv`（约 70MB）→ 拖进转换器 → 用新 `ecdict.js` 替换。**当前 ecdict.js 已含词频；考试标签需要用最新版转换器再跑一次**（若「词汇→词书」提示"没有标签数据"，就是还没跑这一次）。

## 重要注意事项 / 坑
- **Linux 沙盒整个项目期间一直无法启动**（报"下载失败"）。意味着：不能跑 node/lint/浏览器测试，所有代码靠人工逐行核对，**测试要靠用户在浏览器里实际操作**。新对话开工时可以先试一下沙盒是否恢复。
- **不要整篇 Read `ecdict.js`**（会爆上下文）。
- 保持代码风格：原生 ES5 风格、render()/bind() 模式、CSS 写在 index.html。
- 改完告诉用户「刷新 index.html」即可看到（纯前端改动无需重跑词典）。
